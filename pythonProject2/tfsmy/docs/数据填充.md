# main.py 数据填充工作流说明文档

## 📋 概述

`main.py` 是整个“天府市民云”数据采集系统的**一键式总控脚本**。其核心目标是自动化执行所有碎片化的采集模块（爬虫、API提取等），并将分散的数据统一汇总至 `daily` 数据库的各张报表中。

通过运行 `main.py` 一次，即可获取并计算出**昨日（T-1）**的所有关键业务指标。

## 📊 核心业务逻辑说明

### 1. 累计服务总次数汇总 (`update_total_service_times`)
1. **读取昨日增量**：从 `5100_detail` 表中求和，得到昨日所有服务的点击总数。
2. **读取前日累计**：从 `platform_daily_metrics` 表中提取前天的 `total_service_times`。
3. **执行累加**：`昨日累计 = 前日累计 + 昨日增量`。

### 2. 综合指标汇总计算 (`update_daily_aggregates`)
脚本会自动计算并更新 `platform_daily_metrics` 表中的以下关键空缺字段：
- **全平台总日活 (platform_dau)**：
  `platform_dau = 安卓日活 + iOS日活 + 鸿蒙日活 + 支付宝小程序日活 + 微信小程序日活 + 智能前端日活`。
- **累计注册用户 (total_register_users)**：
  `昨日累计 = 前日累计 + 昨日新增注册`。
- **累计实名用户 (total_realname_users)**：
  `昨日累计 = 前日累计 + 昨日新增实名`。

### 3. 全库表自动去重 (`deduplicate_all_tables`)
- **目标**：解决因脚本重复运行或网络重试导致的重复记录问题。
- **机制**：通过临时表技术，识别表中所有字段完全一致的记录，仅保留其中一条，确保数据的唯一性。

## ⚠️ 注意事项与异常处理

1. **日期锁定**：脚本启动时会自动锁定 `target_date = 今天 - 1天`。
2. **环境依赖**：脚本使用 `sys.executable` 确保子模块在与主脚本相同的虚拟环境中运行。
3. **失败汇总**：执行结束后，脚本会打印“失败模块列表”。若某个脚本失败，请查看对应脚本的日志（通常位于下载目录或控制台输出）进行排查。
4. **断点补数**：若 `5100_detail.py` 运行失败，则最后的 `total_service_times` 汇总将无法执行，需手动修复该表数据后再次运行 `main.py`。

## 修复说明
如果13号数据缺失，导致14、15日累加数据错误，16日数据待填写。通过此次 main.py 的修复，代码会一直向前查找数据库中最近一次有效的非空记录（例如12日的累计值）。因此只要16日脚本正常运行，即使之前有缺失，也能根据最新可用记录正确计算出16日的累计值。
